  2. Testbed: A game engine for ScummVM

    3. ScummVM is a highly portable collection of game engine recreations. The ScummVM code can be broadly divided into two parts, the frontend (platform independent, contains the game engine logic) and backend (platform dependent,graphics, sound etc..).
    My task was to write an engine that would interactively guide you through a dummy game and in the meantime, tests almost all the functions provided by the backend API.
    More details about this can be found on ScummVM GSoC'10 page.
    The code for this is available here on Google Code.

  3. TCP/IP Stack using design patterns.

    4. While exploring web-based Operating systems, i came across ES Operating System project. The design of TCP/IP stack of ES fascinated me and I made an attempt to simulate this design independently, in a limited capability. With this understanding of the TCP/IP stack, I went on to implement the support for ARP Caching and ICMP redirect and integrated it to ES operating system.
    More details on this can be found in my major project report.

  4. Distributed Proxy Server

    5. Proxy Servers are common to any academic environment. With the volume of request they handle, it seems logical to distribute the core task on multiple machines. With some ideas from this paper by Tsui, Kaiser, Liu, we had proposed a design which is outlined here in this paper which was presented in an intra-college paper presentation event.
